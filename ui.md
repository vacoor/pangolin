# Pangolin Routing UI 设计文档

## 概述

基于 pangolin-routing 的 INI 配置结构，设计一套 Web 管理界面，涵盖以下核心功能模块：

- **代理服务器**（Proxy）：上游代理服务器的增删改查
- **代理服务器组**（Proxy Group）：代理组合策略配置
- **规则集**（Rule Set）：路由规则文件管理
- **端口监听**（Listen）：多协议端口监听配置
- **TUN 设备**（Tun）：虚拟网卡与 Fake DNS 配置

| 协议 | 必填字段 | 可选字段 |

## 二、代理服务器组（Proxy Group）

对应配置：`[Proxy Group]` 节

```
NAME = combiner, proxy1[, proxy2, ...]
```

### 2.1 列表页

```
┌──────────────────────────────────────────────────────────────────────┐
│ 代理服务器组                                      [+ 添加代理服务器组]│
├──────────────┬───────────┬──────────────────────────────┬────────────┤
│ 名称         │ 策略      │ 成员                         │ 操作       │
├──────────────┼───────────┼──────────────────────────────┼────────────┤
│ VPN-CHAIN    │ chain     │ VPN-INNER-PROXY              │ 编辑 删除  │
│ REMOTE-CHAIN │ chain     │ VPN-REMOTE → VPN-INNER-PROXY │ 编辑 删除  │
│ AUTO-SELECT  │ select    │ VPN-LOCAL, VPN-REMOTE        │ 编辑 删除  │
│ AUTO-FALLBACK│ fallback  │ VPN-LOCAL, VPN-REMOTE        │ 编辑 删除  │
└──────────────┴───────────┴──────────────────────────────┴────────────┘
```

**成员显示规则：**
- `chain`：用 `→` 连接，表示链式顺序
- `select` / `fallback`：用 `, ` 连接

### 2.2 添加/编辑表单

```
┌─────────────────────────────────────────────────┐
│ 添加代理服务器组                             [×] │

│  ○ select    按响应时间自动选择最优代理          │
│  ○ fallback  按顺序尝试，失败时切换到下一个      │
│                                                  │
```

## 三、规则集（Rule Set）

对应配置：`[Rule]` 节 + 规则列表文件（`.list`）+ `[External]` 节

### 3.1 总览

规则分为三种来源：

| 来源 | 配置形式 | 说明 |
|------|----------|------|
| 内联规则 | `DOMAIN-SUFFIX,example.com,UPSTREAM` | 直接写在 `[Rule]` 中 |
| 本地规则集 | `RULE-SET,rule.d/xxx.list,UPSTREAM` | 引用本地 `.list` 文件 |
| 外部规则集 | `[External]` + `RULE-SET,...,UPSTREAM` | 从 URL 拉取的外部规则 |

### 3.2 规则列表页

```
┌──────────────────────────────────────────────────────────────────────┐
│ 路由规则                                        [+ 添加规则] [导入]  │
│                                                                      │
│ 规则优先级说明：规则按列表顺序匹配，命中第一条即停止。              │
├──────┬──────────────────┬────────────┬──────────────┬──────┬────────┤
│ 排序 │ 类型             │ 匹配内容   │ 目标代理     │ 条数 │ 操作   │
├──────┼──────────────────┼────────────┼──────────────┼──────┼────────┤
│  ≡ 1 │ RULE-SET (本地)  │ video.list │ 国外流量     │ 142  │ 编辑 删│
│  ≡ 2 │ RULE-SET (外部)  │ [外部URL]  │ 国外流量     │ 521  │ 编辑 删│
│  ≡ 3 │ DOMAIN-SUFFIX    │ github.com │ DIRECT       │  —   │ 编辑 删│
│  ≡ 4 │ RULE-SET (本地)  │ cn.list    │ DIRECT       │ 8820 │ 编辑 删│
│  ≡ 5 │ MATCH（默认）    │ 所有流量   │ 国外流量     │  —   │ 编辑   │
└──────┴──────────────────┴────────────┴──────────────┴──────┴────────┘
```

**行说明：**
- 第一列 `≡` 可拖拽排序（规则顺序直接影响匹配优先级）
- 最后一行 `MATCH` 为默认规则（兜底），始终置底且不可排序到其他位置
- 外部规则集显示 URL 的主机名缩写

## 五、TUN 设备（Tun）

对应配置：`[Tun]` + `[Fake DNS]` 节

```ini

198.18.0.1/24 = DEFAULT

[Fake DNS]
INET4 = 198.18.0.1/24
INET6 = 2001:2::/48
LEASE-TIME = 60
```

### 5.3 工作原理说明（页内 Info Card）

```
┌──────────────────────────────────────────────────────────────────┐
│ ℹ TUN + Fake DNS 工作原理                                         │
│                                                                  │
│  1. Fake DNS 拦截 DNS 查询，为目标域名分配虚假 IP（如 198.18.x.x）│
│  2. 操作系统将访问该假 IP 的流量路由到 TUN 虚拟网卡               │
│  3. Pangolin 在 TUN 层面拦截流量，还原真实域名后走代理            │
│                                                                  │
│  此模式可透明代理系统内所有应用，无需单独配置各程序的代理。       │
└──────────────────────────────────────────────────────────────────┘
```

## 七、交互规范

### 7.1 名称唯一性校验

- 代理服务器名称、代理服务器组名称在全局范围内不可重复
- 端口号不可重复
- TUN IP 段不可重叠

### 7.2 引用完整性

- 删除代理服务器时，若被代理组引用则弹出确认提示，列出所有引用方
- 删除代理服务器/组时，若被路由规则引用则同样提示
- 删除规则集文件时，确认后同时移除 `[Rule]` 中对应条目

### 7.3 名称规范提示

- 只允许字母、数字、连字符 `-`、下划线 `_`
- 建议全大写以与内置名称（DIRECT、DROP、DEFAULT）风格统一

### 7.4 变更保存流程

```
编辑操作 → 内存中暂存 → 顶部「保存并重载」变为高亮
    ↓
点击「保存并重载」
    ↓
写入配置文件 → 触发重载 → 显示成功/失败 toast
```

未保存时离开页面弹出确认提示。

### 7.5 错误提示

- 表单验证错误：字段下方红色文字
- 端口被占用：保存后服务器端返回错误，Toast 提示
- 代理连通性失败：列表中状态列显示灰色，可悬停查看错误原因

## 八、数据模型（前端视角）

```typescript
// 代理服务器
interface ProxyServer {
  name: string;           // 唯一标识
  scheme: 'ws' | 'wss' | 'ssh' | 'http' | 'https' | 'ss' | 'trojan';
  host: string;
  port: number;
  path?: string;          // ws/wss 路径
  username?: string;      // ssh 用户名
  password?: string;      // ssh/ss/trojan 密码
  alias?: string;         // URI fragment 别名
  available?: boolean;    // 运行时状态
}

// 代理服务器组
interface ProxyGroup {
  name: string;
  combiner: 'chain' | 'select' | 'fallback';
  members: string[];      // 引用 ProxyServer.name 或 ProxyGroup.name
}

// 路由规则（单条）
interface RouteRule {
  type: 'DOMAIN-SUFFIX' | 'DOMAIN' | 'DOMAIN-KEYWORD'
      | 'IP-CIDR' | 'IP-CIDR6' | 'GEOIP'
      | 'RULE-SET' | 'MATCH';
  value?: string;         // 匹配参数（MATCH 无此字段）
  upstream: string;       // 目标代理名称
  order: number;          // 排序权重
}

// 规则集文件
interface RuleSetFile {
  path: string;           // 相对于 conf/ 的路径
  source: 'local' | 'external';
  externalName?: string;  // [External] 中的 key
  externalUrl?: string;
  rules: Array<{ type: string; value: string }>;
}

// 端口监听
interface PortListener {
  port: number;
  protocols: Array<'SOCKS4' | 'SOCKS5' | 'HTTP'>;
  upstream: string;       // 默认代理，'DEFAULT' 表示动态路由
}

// TUN 接口
interface TunInterface {
  address: string;        // CIDR 格式，如 198.18.0.1/24
  upstream: string;       // 默认代理
}

// Fake DNS 配置
interface FakeDnsConfig {
  enabled: boolean;
  inet4?: string;         // IPv4 地址池 CIDR
  inet6?: string;         // IPv6 地址池 CIDR
  leaseTime: number;      // 租约时间（秒）
}
```
